# 判断数据类型

使用typeof和instanceof都不精确，本篇文章介绍如何封装一个使用方便，且易于扩展的类型判断库。

# instanceof和typeof原理

instanceof是判断左侧实例的原型链上是否有右侧构造函数的原型，比如

  function Animal() {
      this.property1 = 'animal'
  }
  function Fish() {
      this.property2 = 'fish'
  }

  function Shark() {
      this.property3 = 'shark'
  }
  function myExtend(Child, Parent) {//寄生组合继承
      function fn(){}
      fn.prototype = Parent.prototype
      Child.prototype = new fn()
      Child.prototype.constructor = Child
  }
  myExtend(Fish, Animal)
  myExtend(Shark, Fish)
  const shark = new Shark()
  console.log(shark)    //{property3: shark}
  console.log(shark instanceof Animal)    //true

# typeof原理

typeof只能对以下类型进行判断

  typeof 1 // number
  typeof 'str' //string
  typeof false // boolean
  typeof undefined //undefined
  typeof function(){}   //function
  typeof Symbol(blabla')   //symbol
  typeof 1234567n  //bigint
  typeof [], typeof null, typeof {} //object

# 封装一个准确的判断数据类型的方法

主要依赖一个方法，就是Object.prototype.toString()，这个方法返回一个字符串，格式为[object typeName] 比如

  Object.prototype.toString.call([])  /*'[object Array]'*/

  function type(target) {
      const str = Object.prototype.toString.call(target)
      return str.match(/^\[object (.*?)\]$/)[1].toLowerCase() // 利用处理字符串 并转换成小写
  }

  const typesArr = ['Null', 'Undefined', 'Object', 'Array', 
                      'Function', 'Regexp', 'Number', 'Boolean', 'String']
  typesArr.forEach( item => {
      type[`is${item}`] = function(target) {
          return type(target) === item.toLowerCase()
      }
  })
