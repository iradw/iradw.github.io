在React中使用TypeScript

DivYang 2020-03-04 JSReact

使用ts编写React是目前前端的一个趋势，ts可以规范代码，尤其是数据类型和接口的规范，更适合大型应用的开发，使代码更加健壮。

# 与js不一样的地方

  1. 生成组件要指定返回类型，比如
const MyComponent: React.FC = () => {
  return (
        <div>组件</div>
  )
}

React.FC就是返回类型：function component

# 数据传递

在js中，父组件传递给子组件直接写就好，当然可以添加prop-types进行传值类型的约束，在ts中就不需要依赖，因为ts中自带一个类型检查-->接口。 在子组件中定义一个接口IProps，这个接口就可以规定，父组件传过来的值，必须符合某种要求。 比如一个导航栏组件，用父组件传递给子组件导航分栏的名字和连接

导航组件(子组件)

  interface IProps {
      navTitles:string[], 
      navLinks:string[]
   }

  const Nav: React.FC<IProps> = (props) => { //添加泛型 约束props
      const [opacity, setOpacity] = useState(0.1)
      useEffect(() => {
          setInterval(() => {
              console.log('aaa')
              setOpacity(v => v+0.1)
          }, 1000)
          console.log(opacity)
      }, [])
      const {navLinks, navTitles} = props

      return (
          <nav className="nav" style={{opacity}}>
              <ul className="nav-list">
                  {
                      navLinks.map((item, index) => 
                          <li className="nav-item" key={index}>
                              <Link to={item}>{navTitles[index]}</Link>
                          </li>
                      )
                  }
              </ul>
          </nav>
      )
  }

主页组件(父组件)

  const Parent: React.FC = () => {
      const navProps = {
          navLinks: ['/', 'link2', 'link3', 'link4'],
          navTitles: ['首页', '导航2', '导航3', '导航4']
      }
      return (
          <Router>
              <Nav {...navProps}></Nav>
              <Content></Content>
          </Router>
      )
  }

如果父组件中navProps这个对象里面没有navLinks和navTitles这两个属性，就会报出警告。 如果是用class的方式创建组件，接口生命是一样的，泛型约束这样写

  class Parent extends React.Component<IProps, IState>
1

IProps为组件的props约束，IState为组件的state约束

# 状态管理

hooks中的写法是给useState指定泛型类型，比如

  const [welcome, setWelcome] = useState<string>('欢迎欢迎')
1

这样welcome是string类型的，setWelcome的参数也必须为strng类型

class中的写法和数据传递时一样，写一个接口IState，让组件的state实现这个接口，也就介绍数据传递最后写的那行代码。
