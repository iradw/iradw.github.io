# Jest中的匹配器

# 真假匹配器

# toBe

toBe表示完全相等，内容相同的两个对象，类似===，是不能通过toBe的，比如

test('测试', () => {
  expect({key1: 'value1'}).toBe({key1: 'value1'}) //不通过
})
1
2
3

# toEqual

toEqual表示相等，类似==，上述情况是可以通过toEqual的

test('测试', () => {
  expect({key1: 'value1'}).toEqual({key1: 'value1'}) //通过
})
1
2
3

# toBeNull、toBeUndefined、toBeDefined

toBeNull只匹配null值，toBeUndefined只匹配undefined，toBeDefined可以匹配除了undefined的所有值，包括0、null。

# toBeTruthy、toBeFalsy

truthy表示真值，只要是if语句中会转为true的均为truthy值，如[] {} 1 -1

# 数字匹配器

# toBeGreaterThan

比某个值大

test('测试', () => {
   expect(15).toBeGreaterThan(10) // 通过
})
1
2
3

# toBeCloseTo

对于比较浮点数相等，使用toBeCloseTo 而不是toEqual，因为你不希望测试取决于一个小小的舍入误差。

test('两个浮点数字相加', () => {
  const value = 0.1 + 0.2;
  //expect(value).toBe(0.3);           这句会报错，因为浮点数有舍入误差
  expect(value).toBeCloseTo(0.3); // 这句可以运行
});

# 字符串

# toMatch

匹配字符串，可是使用正则。

test('测试', () => {
  // expect('javaScript').toMatch('java');
  expect('javaScript').toMatch(/SCRIPT/gi); // 两个都可通过

# 数组

# toContain

检查数组(可以是Set)中是否包含某个值，很常用。

test('测试', () => {
  expect(['js', 'java', 'go']).toContain('js'); // 通过
})

# 异常

# toThrow

检查某个方法运行时是否发生异常，expect方法需要接受一个方法引用。

function thrower() {
  throw new Error('随便抛出的一个异常')
}
test('测试', () => {
  expect(thrower).toThrow()  // 通过
})
test('测试', () => {
  expect(thrower).toThrow('不随便')  // 不通过
})
test('测试', () => {
  expect(thrower).not.toThrow()  // 不通过
})
