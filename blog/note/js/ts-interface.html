TypeScript的接口与泛型

DivYang 2020-02-08 TS

# TypeScript的接口与泛型

# 属性类型接口

对函数的参数进行检查，如果有一个方法printName()要求参数是一个对象，对象中必须包含firstName和lastName两个字符串类型的属性，age属性是可选的数值类型，则可这样写

interface nameOption {
    firstName: string,
    lastName: string,
    age?: number
}
function printName(name: nameOption){
    console.log(name.firstName + ' ' + name.lastName)
}

# 函数类型接口

对方法传入的参数以及返回值进行约束

interface IPrintName{
    (firstName:string, lastName:string):string
}

这个接口可以理解为一个函数模板，参数必须为两个字符串，返回值也是一个字符串。注意：如果想让函数实现函数接口必须按照下面这样进行赋值的函数定义方法，这样才能既指定函数接口，又指定函数返回值。

const prrintName:IPrintName = (firstName:string, lastName:string):string => firstName + ' ' + lastName

也就是说接口是用来规范方法定义的，他可以批量约束；方法里面的参数类型是用来规范方法调用的。

# 类类型接口

这个类类型接口起始就是上面两种接口的整合，类里面有属性和方法。

interface IAnimal{
     name: string,
     eat(food: string): string
}

这样创建一个类，使用implements实现接口，这个类里面就必须用name属性和eat方法，其中eat方法必须是传入字符串参数返回字符串。

class Dog implements IAnimal{
    name: string
    constructor(name:string){
          this.name = name
    }
    eat(food:string){
          return this.name+'吃'+food
    }
}
const dog = new Dog('旺财')

若使用extends继承接口，则该类可以扩展接口

interface IPerson extends IAminal{
       work(workName:string): string
}
class Human implements IPerson{
      name: string
      constructor(name:string){
          this.name = name
     }

    eat(food:string){
           return this.name+'吃'+food
    }

    work(workName:string){
          return this.name+'正在'+workName
    }
}

const XiaoMing = new Human('小明')
XiaoMing.eat('米饭')  //小明吃米饭
XiaoMing.work('敲代码')  //小明正在敲代码

# 泛型函数

下面这段代码

function foo<T>(n1:T, n2:T): T {
    return n1
}

表示foo方法执行的时候需要指定一个类型T，两个参数都是T，返回值类型也是T，

foo<number>(1, 2)  //合法调用
foo<number>('1', 2) //不合法调用
foo<string>('1', '2') //合法调用

# 泛型类

和泛型函数同理，只是在定义类的时候使用<T>

class MyList<T> {
    private list: T[] = []
    add(value: T):void {
        this.list.push(value)
    }

    max():T {
        let max:T = this.list[0]
        for (let i = 1; i < this.list.length; i++) {
            let item: T = this.list[i]
            if (item > max) {
                max = item
            }
        }
        return max
    }
}
    let myList2 = new MyList<string>()
    myList2.add('a')
    myList2.add('h')
    myList2.add('b')
    alert(myList2.max()) //h

# 泛型接口

之前的函数泛型只能返回string类型的值，不够灵活

interface fooOption{
    (v1:string, v2:string):string
}

进行如下改动

interface fooOption{
    <T>(v1:string, v2:T):T
}

const foo:fooOption = function<T>(v1:string, v2:T):T {
    return v2
}
foo<string>('', '')

给接口中的函数模板添加泛型，可以灵活的返回不同类型。
