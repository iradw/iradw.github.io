# Docker笔记

# Docker安装

# 1. 安装Docker

本人环境为ArchLinux，因此安装非常简单，sudo pacman -Sy docker即可，其他环境可查询其他文档，也比较简单，只不过需要重新设置镜像源。

# 2. 启动Docker

systemctl start docker启动，systemctl enable docker设置开机自启动。

# 3. 更换镜像源

docker默认使用官方镜像源，速度慢，需要更换源

  1. 在/etc/docker下，创建daemon.json文件，写入下面的代码，更换为中科大的镜像源。
{
  "registry-mirrors": ["https://docker.mirrors.ustc.edu.cn"]
}

  1. 重启docker服务，systemctl restart docker
  2. 检测是否成功，执行docker run hello-world，如果出现证明docker安装和配置成功。

# Docker中央仓库

  1. 官方仓库，https://hub.docker.com，下载速度非常慢
  2. 推荐，https://hub.daocloud.io

# Docker镜像操作

  1. 拉取镜像 在daocloud网站上搜索需要的镜像，比如nginx，在版本一栏，点击拉取可以得到镜像名，即daocloud.io/library/nginx:latest，执行docker pull daocloud.io/library/nginx:latest即可拉取nginx镜像。
  2. 查看本地镜像 docker images，IMAGE_ID就是镜像的唯一标识
  3. 删除本地镜像 docker rmi 08393e824c32，08393e824c32是刚刚下载的NGINX镜像的IMAGE_ID。

# Docker容器操作

  1. 运行容器 docker run 镜像的标识|镜像的名称[:镜像的TAG]，这种方法比较简单，但是不能指定端口，需要用到下面的方式启动。

  2. 常用参数 docker run -d -p 宿主机端口:容器端口 --name 容器名称 镜像的标识|镜像的名称[:镜像的TAG]

    -d代表后台运行，-p宿主机端口为我们需要在浏览器访问的端口，容器端口就是镜像在虚拟环境中运行的端口,

    如docker run -d -p 8083:80 --name nginxContainer 083，执行后在本地浏览器访问localhost:8083就可以看到NGINX的界面。填写镜像标识时不需要完整填写，只需填写前几个字符，并保证唯一即可。

  3. 查看容器 docker ps -qa -q查看容器id，-a查看全部容器，-qa查看全部容器id

  4. 查看容器日志 docker logs -f 容器id|容器名

  5. 进入容器内部 docker exec -it 容器id|容器名 bash 输入exit退出容器

  6. 停止容器 docker stop 容器id|容器名 docker stop $(docker ps -qa)停止全部容器

  7. 启动容器 docker start 容器id|容器名

  8. 删除容器 docker rm 容器id|容器名 docker rm $(docker ps -qa)删除全部容器

    删除容器前需要停止容器。

  9. 把宿主机的文件复制到docker容器内部 docker cp 文件名 容器id 容器内部路径
