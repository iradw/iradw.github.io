NGINX的学习

DivYang 2020-01-16 NGINX

# NGINX是前端必会的服务器，是高性能的http及反向代理服务器，它有性能高，占用内存小等优点。记录nginx的学习过程以及使用。

# 正向代理

正常情况下，国内不能够访问youtube的官网，如果想看youtube的视频，那就可以在客户端配置一个代理服务器，这样我们可以访问代理服务器，告诉代理服务器我要看youtube，代理服务器访问youtube，把youtube的页面返回值我，这样我就看到了youtube。

# 反向代理

正向代理需要客户端进行配置，而反向代理客户端0配置，当客户端访问反向代理服务器时，由该服务器去访问目标服务器，再返回给客户端，此时反向代理服务器和目标服务器对外就是一个服务器，可以隐藏真实服务器的ip地址，也方便负载均衡。

# 负载均衡

一般的项目都会用到负载均衡，一台服务器处理不了的并发请求，NGINX代理服务器可以将这些请求平均分布到配置的多台服务器中。

# 动静分离

将服务器分成动态资源服务器(jsp, servlet)和静态资源服务器(HTML，CSS，JS)，其实和负载均衡的概念差不太多，就是按文件类型请求服务器的意思。

# NGINX常用命令

  1. 查看nginx版本 nginx -v
  2. 启动nginx nginx 查看nginx是否启动可以看linux的进程 ps -ef | grep nginx
  3. 关闭nginx nginx -s quit
  4. 重新加载nginx nginx -s reload

# NGINX配置文件

nginx配置文件的位置 /etc/nginx/nginx.conf和/etc/nginx/conf.d/defalult.conf

  1. nginx配置文件由三部分组成

  • 全局块 配置整体运行的指令 user nginx; worker_processes 1; #nginx 处理并发的数量 error_log /var/log/nginx/error.log warn; pid /var/run/nginx.pid;

  • events块 配置服务器与用户的网络连接

    这部分对NGINX性能影响较大 events { worker_connections 1024; }

  • http块

     http {
 include       /etc/nginx/mime.types;
 default_type  application/octet-stream;
 
 log_format  main  '$remote_addr - $remote_user 
 [$time_local] "$request" '
 '$status $body_bytes_sent 
 "$http_referer" '
 '"$http_user_agent" "$http_x_forwarded_for"';

  access_log  /var/log/nginx/access.log  main;

  sendfile        on;
  #tcp_nopush     on;

  keepalive_timeout  65;

  #gzip  on;

  include /etc/nginx/conf.d/*.conf; # 这里表示以模块方式引入server配置，server配置在/etc/nginx/conf.d/*.conf下
 }

  • serve块 /etc/nginx/conf.d/*.conf文件

    这部分是配置最频繁的部分，一般一个文件代表一个代理配置。

# 配置反向代理

打开/etc/nginx/conf.d/defalult.conf文件 

listen 80 # nginx监听的端口号
server_name # 服务器域名可以是localhost或者填写服务器的ip地址
location / {   # 拦截 / 请求
    proxy_pass http://127.0.0.1:3000; # 将请求发送至该IP端口下 也可以是其他服务器的ip
}

将3000端口的项目启动，再将nginx启动，这样就做到了在浏览器访问80端口，nginx将请求转发到了3000端口，也就配置好了反向代理。

# 实例：next.js项目的部署

1. 后端使用eggjs开发，eggjs自带一个进程守护，因此部署时按照官网的描述就可以。
2. 前台展示使用react服务端渲染框架next.js开发，将本地除了node_modules文件夹上传至服务器，先安装运行依赖`cnpm install --production`
   运行项目，使用pm2守护进程 `pm2 start npm --name 'iradw-blog' -- run start` 项目会在3000端口运行
3. 后台管理使用react开发，在`/etc/nginx/conf.d/default.conf` 文件中做刚刚写过的配置
   新建`admin.conf`文件，可以直接复制default.conf文件再进行修改，修改如下

server {
    listen 8080;
    server_name: localhost;
    location / {
        root /blog/admin/build
        index index.html index.htm
    }
}

这样就实现了在浏览器中输入ip地址访问前台，输入ip:8080访问后台管理。

# 负载均衡的配置

在配置文件http块中，写入服务器列表
upstream myserver {
      server 192.168.1.1:3000;
      server 192.168.1.2:3000;
}

在/etc/nginx/conf.d中的配置文件中配置

server {
    listen 80;
    server_name localhost;
    location / {
        proxy_pass http://myserver;
        root html;
        index index.html index.htm
    }
}

当启动nginx后，访问nginx所在的服务器，nginx会把请求均衡转发到配置的服务器列表中。

# nginx服务器分配策略

  1. 轮询(默认)

每个请求按时间顺序逐一分配到不同的后端服务器，如果某个后端服务器宕机，nginx可以自动剔除。 2. weight权重

weight代表权重，默认值为1，权重越高被分配的客户端越多。 upstream myserver { server 192.168.1.1:3000 weight=1; server 192.168.1.2:3000 weight=2; } 3. ip_hash 每个请求按访问的hash结果分配，这样每个方可固定访问一个后端。 upstream myserver { ip_hash server 192.168.1.1:3000; server 192.168.1.2:3000; } 4. fair(第三方)

按后端服务器的响应时间来分配请求 upstream myserver { server 192.168.1.1:3000; server 192.168.1.2:3000; fair }
