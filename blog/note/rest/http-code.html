http常见状态码

DivYang 2020-01-15 HTTP

# 2xx

状态码2xx表示请求成功处理。

# 200 OK

这是我们最喜欢的状态码，表示前后端都没问题，请求成功。

# 204 NO Content

开发时，使用fetch发送请求，会有两个响应，第一个响应不带任何内容，只有一个204状态码，第二个才是我们想要的服务器返回的内容。

当发送一个跨域的可能对服务器数据产生副作用的请求(post、put等)时，需要先发送一个预检请求，如果服务端允许访问则返回204，之后发起实际的http请求。在预检请求的返回中，服务器端也可以通知客户端，是否需要携带身份凭证（包括 Cookies 和 HTTP 认证相关数据）。

# 3xx

请求需要附加操作，如重定向。

# 301 Moved Permanently

301 Moved Permanently 
...
Location:`b.com`
...

//上述内容表示：请求的资源被永久转移至b.com，之后建议直接请求b.com

# 302 Found

301 Moved Permanently 
...
Location:`b.com`
...
// 和301类似，只不过302是临时转移，301是永久转移

# 303 See Other

和302类似，303除了提示客户端去请求Location外，还会提示客户端要用get方法去访问Locattion。

# 304 Not Modified

表示资源未改变，直接使用缓存。也就是说，服务器里面的内容和你上一次请求时是一样的，没必要在发送一次了，你直接使用浏览器的缓存就可以了

# 4xx

客户端发生错误，也就是前端程序员的问题。

# 400 Bad Request

这是开发时最容易出现的错误，可能是请求有语法错误。

  • 请求体字段和服务端需要的字段不匹配。
  • 请求方法错误，比如后端规定put，前端用了post。
  • url地址写错了

# 401 Unauthorized

当我们去请求其他平台提供的开放接口时，如bilibili、腾讯云等需要提供身份验证，把Authorization放在请求头中，如果没有，就会返回401。

# 403 Forbidden

没有权限访问该资源，具体原因会在错误中给出。

# 404 Not Found

很常见，要么时服务端没有资源，要么是前端uri写错了。

# 5xx

此类错误大多为服务端错误。

# 500 Internal Server Error

后端代码有问题。

# 502 Bad Gateway

如果客户端不是直接请求的服务端，而是请求的代理，当代理服务器收到真正服务器的无效响应时，会给客户端返回502。

# 503 Service Unavailable

服务器不可用

# 504 Gateway Timeout

访问超时，可以检查一下是不是自己的网断了。
